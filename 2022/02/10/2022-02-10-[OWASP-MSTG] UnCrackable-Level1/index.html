🤔 UnCrackable 이란?


OWASP에서 만든 취약한 모바일 애플리케이션이며, Android뿐만 아니라 iOS용도 있습니다.


[OWASP-MSTG Crackmes]

GitHub




🔎 Solve


진단 환경


애플리케이션 : UnCrackable-Level1.apk

기기 : 삼성 갤럭시 S7 (안드로이드 8.0), (루팅 O)

기타 : Frida (12.6.11), jadx (1.3.2)



애플리케이션 설치


> adb install UnCrackable-Level1.apk




애플리케이션 분석


[ 애플리케이션 실행 ]


해당 애플리케이션을 실행하면 루팅 감지를 하고 OK 버튼을 누르면 애플리케이션이 꺼집니다.



[ 루팅 우회 소스코드 확인 ]

jadx로 해당 애플리케이션을 디컴파일하여 루팅탐지하는 부분의 코드를 확인해보았습니다.



루팅 탐지하는 문구의 내용을 jadx의 텍스트 검색 기능을 사용하여 해당 코드를 확인해보았습니다.



해당 코드를 보면 onCreate 메소드 내부에 a 메소드를 호출하면서 루팅탐지를 하고있습니다.

a 메소드 부분을 따라가서 확인해보았습니다.


OnClick 메소드가 작동하면 System.exit(0);이 작동되는 것을 볼 수 있습니다.



[ 후킹 ]

Frida를 사용해 System.exit(0); 부분을 후킹하여 OK 버튼을 눌러도 해당 코드 작동이 안되도록 재작성해보겠습니다.



해당 자바스크립트 파일을 만들어 실행시켜주었습니다.




OK 버튼을 눌러주니 console.log에 Root Bypass success 문자가 나오고 루팅 탐지를 우회하고 있는 것을 볼 수 있습니다.



[ Secret String 후킹 ]

Secret String 넣는 곳에 아무 문자나 넣고 VERIFY 버튼을 눌렀더니 틀렸다는 알림창이 나왔습니다.




루팅 탐지 후킹과 마찬가지로 jadx 텍스트 검색 기능을 이용하여 알림창에 뜬 문자를 입력해주었습니다.




해당 코드를 보면 MainActivity 클래스 내부에 verify 메소드가 있고 그 안에 Nope... 이라는 값이 있습니다.

우리가 입력한 Secret String은 obj 안에 들어가는 것을 볼 수 있습니다. 해당 값은 if 문 a 클래스 내부에 있는 a 메소드 인자값으로 들어가고 있습니다.

해당 if 문을 TRUE로 바꾸어주면 성공이라는 메시지가 나옵니다.



a 클래스의 a 메소드를 후킹하여 재작성해주었습니다.


해당 자바스크립트 파일을 만들어 실행시켜주었습니다.




아무 문자나 넣고 VERIFY 버튼을 눌렀더니 성공했다는 알림이 떴습니다.
