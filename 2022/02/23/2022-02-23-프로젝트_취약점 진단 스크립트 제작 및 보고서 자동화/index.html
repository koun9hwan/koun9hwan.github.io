[프로젝트] 취약점 진단 스크립트 제작 및 보고서 자동화

Posted on 2022-02-23 In Project

🧶 프로젝트 개요


KISA의 주요정보통신기반시설 기술적 취약점 분석 평가 방법 상세가이드를 통하여 리눅스 서버의 취약점을 학습하고, 쉘 스크립트를 제작하였습니다.
또한 해당 스크립트를 가지고 취약점 진단 보고서를 자동화 하였습니다.



해당 프로젝트를 한 이유?


주요정보통신기반시설 기술적 취약점 분석 평가 방법 상세가이드를 통하여 서버의 취약점을 공부할 수 있고, 직접 쉘 스크립트를 만들어보면서 해당 취약점을 자동으로 진단할 수 있다고 생각되어 이 프로젝트를 하게 되었습니다.



프로젝트 진행 기간 & 일정

gantt

dateFormat  YYYY-MM-DD

section 진행 기간

취약점 공부 :done,  des1, 2021-12-20,2022-01-03

쉘 스크립트 제작:active, des2, 2022-01-04,2022-01-18

자동화를 위한 쉘 스크립트 수정:         des3, 2022-01-19,2022-01-26

보고서 틀 작성 및 자동화 스크립트 제작:active, des4, 2022-01-27,2022-02-03

프로젝트는 2021년 12월 20일부터 2022년 2월 3일까지 진행되었으며 총 6주차 진행하였습니다.

0 ~ 2주차 : 주요정보통신기반시설 기술적 취약점 분석 평가 방법 상세가이드를 통해 리눅스 서버의 취약점을 공부하였습니다.

2 ~ 4주차 : 취약점 72개의 항목에 대해 취약점을 진단할 수 있는 쉘 스크립트를 제작하였습니다.

4 ~ 5주차 : 자동화 보고서를 위한 취약점 진단 쉘 스크립트 출력 결과를 수정/보완 하였습니다.

5 ~ 6주차 : 기본적인 보고서 틀 작성 및 자동화 스크립트를 제작하였습니다.



프로젝트 진행 환경


운영체제

취약점 진단 대상은 CentOS Linux release 7.8.2003 (Core) 를 사용하였습니다.


도구

코드를 작성하는 도구로는 vi와 Visual Studio Code를 사용하였습니다.
작성한 코드와 보고서를 옮기는 도구로는 WinSCP를 사용하였습니다.


언어

사용 언어로는 Shell Script, Python3를 사용하였습니다.

💉 보고서 틀 작성 및 자동화 스크립트 제작


자동화 보고서를 위한 기본적인 보고서 틀 및 스크립트를 제작하였습니다.



보고서



보고서의 sheet 1번 입니다.

취약점 진단 결과를 한눈에 볼 수 있도록 취약점 항목 개수 항목별 개수 항목 주요도별 개수 담당자 확인 취약점 리스트를 차트와 표로 정리하였습니다.




보고서의 sheet 2번 입니다.

취약점 진단 결과의 항목을 상세하게 볼 수 있는 표와 항목별 양호/취약 판단 기준 표를 만들었습니다.




보고서의 sheet 3번 입니다.

해당 시트에서는 취약점 진단 결과가 자동으로 입력되는 시트입니다.



자동화 스크립트


취약점 진단 쉘 스크립트의 결과를 자동으로 엑셀에 삽입시켜주는 자동화 스크립트 파일을 파이썬으로 제작하였습니다.



자동화를 하는 방식은 먼저 쉘 스크립트의 결과를 csv로 출력해줍니다.

출력한 csv 파일을 xlsx(엑셀) 파일로 변환 후 그 값을 보고서의 3번 시트에 삽입해주는 방식입니다.



취약점 진단 후 나온 결과물 입니다. 차트와 표가 자동으로 출력됩니다.



🤔 부족한 점 & 보완할 점


취약점 항목 별 현재 현황을 출력해주어 담당자가 취약/양호 판단이 쉬워 질 수 있도록 만들면 좋을 거 같습니다.

예) 불필요한 계정 제거 → 현재 계정 목록 출력



항목 별(상, 중, 하) 점수를 부여하여 해당 시스템이 몇 점인지 한눈에 볼 수 있도록 차트와 최종 점수를 보고서에 표현해주면 좋을 거 같습니다.
